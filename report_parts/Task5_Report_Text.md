# Task 5: Pre-trained Language Models

For Task 5, we fine-tuned a pre-trained transformer model for financial sentiment classification. We used `distilbert-base-uncased`, a smaller and faster version of BERT. DistilBERT is useful for this project because it keeps the main idea of contextual transformer representations while being more practical to train on a laptop than full BERT.

In contrast to the classical models from Task 3, the transformer model did not use Bag-of-Words or TF-IDF features. Instead, the original sentence text was passed to the DistilBERT tokenizer. The tokenizer split words into subword units, added the required transformer inputs, padded or truncated sentences to a maximum length of 64 tokens, and produced the numerical input needed by the model.

The model was fine-tuned in both the multiclass and binary settings. For the multiclass setting, we used the three sentiment labels negative, neutral, and positive. For the binary setting, we removed the neutral class and trained only on negative and positive examples. In both cases, we used an 80/20 stratified train-test split, one training epoch, a batch size of 16, a learning rate of 2e-5, a maximum sequence length of 64, and the MPS device on the Mac when available.

In the multiclass setting, DistilBERT achieved an accuracy of 0.8103 and a macro F1 score of 0.7954. This was clearly better than the best classical model from Task 3, which was Naive Bayes with Bag-of-Words. That model achieved an accuracy of 0.7134 and a macro F1 score of 0.6519. The improvement suggests that contextual transformer representations are better suited to this task than simple word-count features.

In the binary setting, DistilBERT achieved an accuracy of 0.9518 and a macro F1 score of 0.9442. This was also clearly better than the best binary Task 3 model, Naive Bayes with Bag-of-Words, which achieved an accuracy of 0.8071 and a macro F1 score of 0.7734. As in Task 3, the binary setting was easier than the multiclass setting because the model no longer had to distinguish neutral business statements from positive or negative sentiment.

The exact transformer scores can vary slightly between runs because neural network training contains some randomness, especially when using hardware acceleration. The transformer model likely performed better because it can use context more effectively. For example, classical models treat words mainly as separate features, while DistilBERT can represent a word differently depending on the surrounding sentence. This is important in financial sentiment analysis because words such as "profit", "sales", or "growth" are not always positive by themselves. Their meaning depends on whether they are connected to increases, decreases, expectations, losses, or other financial context.

However, the transformer model also has disadvantages. It requires more computation, needs additional packages, downloads a pre-trained model, and takes much longer to train than Naive Bayes. The classical models from Task 3 are faster, easier to understand, and easier to reproduce. Therefore, the transformer model gives better performance, but the improvement comes with higher computational cost and more implementation complexity.

Overall, the Task 5 results show that fine-tuning a pre-trained transformer model substantially improves sentiment classification performance on this dataset. The best overall model was DistilBERT in the binary setting. The best multiclass model was also DistilBERT. Compared with Task 3, the transformer model handled the financial language better, probably because it uses contextual representations instead of only isolated word counts.
